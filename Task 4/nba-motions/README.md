# Задача 4. Генерация карт траекторий игроков NBA

[← К Task 4](../README.md) · [К главному README](../../README.md)

Курсовой проект по глубокому обучению. Цель: построить генеративную модель,
которая создает реалистичные траектории движения игроков NBA, и оценить качество
через Inception Score (IS) и Frechet Inception Distance (FID).

Данные: публичный дамп оптического трекинга SportVU сезона NBA 2015/16
(sealneaward/nba-movement-data). Мяч и 10 игроков, 25 кадров в секунду, координаты
в футах на площадке 94 на 50.

## Два подхода (сравнение)

Мы сравниваем два принципиально разных способа генерации траектории:

1. Наш DCGAN - пиксельный подход. Траектория рендерится в картинку 64x128 RGB,
   сверточный GAN генерирует такие картинки напрямую. Ноутбук `final-gan.ipynb`,
   описание в `README_final_gan.md`, схема в `gan_architecture.png`.
2. Coordinate-VAE - координатный подход. Траектория моделируется как
   последовательность из 50 точек, вариационный автоэнкодер учит латентное
   пространство и генерирует координаты, которые потом рендерятся в линию. Ноутбук
   `GAN_VAE_final_notebook.ipynb`, описание в `README_notebook2_coordinate_models.md`,
   схема в `vae_architecture.png`.

## Таблица сравнения

| Критерий | Наш DCGAN | Coordinate-VAE |
| --- | --- | --- |
| Представление данных | Изображение (пиксели) | Координаты (50 точек) |
| Что генерирует | Картинку 64x128 сверткой | Последовательность точек, потом рендер |
| Тип модели | GAN (состязательная) | VAE (автоэнкодер с латентом) |
| Дискриминатор | Да | Нет |
| Связность траекторий | Средняя (за счет рендеринга) | Высокая (по построению) |
| Сложность обучения | Высокая | Низкая |
| Метрики | IS 2.24, FID 37.9 | recon MSE ~5e-4, координатные метрики |

Две парадигмы дополняют друг друга. Пиксельный DCGAN дает реалистичную форму и
лучший FID, но связность не гарантирована и обеспечивается рендерингом.
Координатный VAE строит линию из точек, поэтому она связна по построению, учится
стабильнее, но оценивается координатными метриками, а не FID.

IS и FID тут приближение: InceptionV3 обучена на фотографиях, а у нас штриховые
рисунки. Главный критерий качества - визуальное сходство с реальными траекториями.

Ключевой вывод нашего подхода: качество упиралось не в размер модели, а в
представление данных. Толстая сглаженная линия на белом фоне (без схемы корта)
сделала задачу решаемой для пиксельного GAN, тогда как тонкая линия поверх корта
была нерешаема.

## Структура репозитория

```
final-gan.ipynb                          <- наш финальный ноутбук (DCGAN)
README_final_gan.md                      <- описание нашего ноутбука
gan_architecture.png                     <- схема DCGAN для презентации

GAN_VAE_final_notebook.ipynb             <- второй ноутбук (берем часть Coordinate-VAE)
README_notebook2_coordinate_models.md    <- описание второго ноутбука
vae_architecture.png                     <- схема Coordinate-VAE для презентации

results_bundle.zip                       <- результаты нашего GAN (чекпоинт, метрики, картинки)

data/
  pipeline.py        <- JSON + events.csv -> 64x128 RGB PNG (рендер без корта)
  build_demo.py      <- сборка небольшого demo-набора + zip для Kaggle
  dataset.py         <- PyTorch Dataset
models/
  generator.py       <- DCGAN генератор
  discriminator.py   <- DCGAN дискриминатор
  gan.py             <- обертка (Adam, BCE + label smoothing, опц. hinge/spectral)
utils/
  visualization.py   <- сетки и сравнение real vs fake
  metrics.py         <- IS + FID через InceptionV3
train.py             <- локальный CPU smoke
generate.py          <- генерация PNG из чекпоинта
evaluate.py          <- IS + FID + compare-grid на тесте
_build_nb.py         <- генератор учебного ноутбука
splits/              <- train/val/test (списки путей)
```

Примечание: ноутбук `GAN_VAE_final_notebook.ipynb` содержит несколько моделей
(включая baseline), но в итоговое сравнение мы берем из него только Coordinate-VAE.

## Как воспроизвести (наш DCGAN)

1. Собрать набор картинок: `python data/build_demo.py ... --zip`. Залить zip как
   Kaggle Dataset.
2. Открыть `final-gan.ipynb` на Kaggle, подключить датасет, выбрать GPU и включить
   Internet (нужен для весов InceptionV3).
3. Сначала `OVERFIT = True` (проверка на 200 картинках), затем `OVERFIT = False`
   для полного прогона. Результаты сохранятся в `results_bundle.zip`.

Подробности по первому ноутбуку - в `README_final_gan.md`, по второму - в
`README_notebook2_coordinate_models.md`.
