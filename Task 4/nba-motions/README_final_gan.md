# Финальный ноутбук: генерация карт траекторий NBA через DCGAN

Описание ноутбука `final-gan.ipynb` - наш основной подход к Задаче 4.

## 1. Назначение

Ноутбук обучает GAN, который генерирует карты траекторий движения игроков NBA.
Подход - image-based: траектория рендерится в картинку 64x128 RGB, и сверточный
GAN (DCGAN) учится генерировать такие картинки напрямую в пространстве пикселей.
Одна картинка = путь одного атакующего игрока за один эпизод.

Это альтернатива координатному подходу - Coordinate-VAE (см.
`README_notebook2_coordinate_models.md`), где траектория моделируется как
последовательность координат. Мы сравниваем два подхода: пиксельный GAN и
координатный VAE.

## 2. Данные и предобработка

Источник: публичный дамп оптического трекинга SportVU сезона NBA 2015/16
(sealneaward/nba-movement-data). Мяч и 10 игроков, 25 кадров в секунду, координаты
в футах на площадке 94 на 50.

Шаги предобработки (код в `data/pipeline.py`):

1. Фильтрация эпизодов по EVENTMSGTYPE: берем 1 (Made Shot), 2 (Missed Shot),
   4 (Rebound), 5 (Turnover), 6 (Foul). Служебные и статичные типы (штрафной,
   нарушение, замена, тайм-аут, спорный, удаление, начало и конец периода,
   видеоповтор) отбрасываем.
2. Отсев мусора по длине: только эпизоды от 7 секунд (175 моментов при 25 Hz).
3. Отсев топтания на месте: убираем траектории с размахом движения менее 5 футов.
4. Зеркалирование: целевую корзину определяем по средней x мяча за последние 25
   моментов. Если атака шла влево, отражаем x -> 94 - x. После этого все атаки
   направлены вправо.
5. Рендер 64x128 RGB: белый фон без схемы корта, одна толстая сглаженная линия.
   Рисуем в 4x разрешении и уменьшаем фильтром LANCZOS (плавный градиент вместо
   рваной тонкой линии). Толщина линии 3 пикселя, маркера конца нет.
6. Нормализация пикселей в [-1, 1] (под выход Tanh генератора).
7. Сплит по матчам (game_id), чтобы эпизоды из теста не утекали в обучение.

## 3. Архитектура (DCGAN)

Генератор (около 3.97M параметров):

```
z(100) -> Linear -> reshape 256 x 8 x 16
       -> ConvTranspose s2, BN, ReLU -> 128 x 16 x 32
       -> ConvTranspose s2, BN, ReLU -> 64 x 32 x 64
       -> ConvTranspose s2, Tanh     -> 3 x 64 x 128
```

Дискриминатор (около 0.18M параметров):

```
3 x 64 x 128 -> Conv s2, LeakyReLU       -> 32 x 32 x 64
             -> Conv s2, BN, LeakyReLU   -> 64 x 16 x 32
             -> Conv s2, BN, LeakyReLU   -> 128 x 8 x 16
             -> Flatten -> Linear -> Sigmoid -> P(real)
```

Инициализация весов по DCGAN: свертки N(0, 0.02), BatchNorm N(1, 0.02).

## 4. Обучение

- Лосс: бинарная кросс-энтропия (BCE) с one-sided label smoothing (метка реальных
  0.9, поддельных строго 0). Лосс генератора не-сатурирующий.
- Оптимизатор: Adam, beta1 = 0.5. TTUR: lr генератора 2e-4, lr дискриминатора 1e-4.
- Соотношение шагов: 1 шаг генератора на 1 шаг дискриминатора.
- Обучение считается в шагах, а не эпохах. Перед полным прогоном делаем тест на
  переобучение (overfit gate) на 200 картинках: если генератор не заучивает их в
  узнаваемые траектории, дальше идти нет смысла.
- preload: для небольших наборов загружаем весь набор одним тензором в память GPU,
  это убирает накладные расходы DataLoader и ускоряет обучение.
- Опционально доступны hinge loss и spectral norm (флаги), но в финале используется
  простой BCE с TTUR.

## 5. Метрики

- Inception Score (IS) на 1000 сгенерированных картинок, splits = 10.
- FID между признаками InceptionV3 (pool, 2048-d) реальных тестовых и
  сгенерированных картинок.

InceptionV3 обучена на фотографиях, у нас штриховые рисунки, поэтому IS и FID тут
приближение. Главная проверка качества - визуальное сходство.

Финальный результат нашего GAN: IS = 2.244, FID = 37.9 (на 1000 тестовых картинок).

## 6. Структура ноутбука по секциям

1. Настройка (импорты, seed, устройство).
2. Данные (поиск датасета, Dataset, просмотр реальных примеров).
3. Генератор.
4. Дискриминатор.
5. Лосс и оптимизаторы.
6. Функция обучения с живыми графиками.
7. Запуск (тумблеры OVERFIT, N_SUBSET, LOSS, SPECTRAL).
8. Кривые обучения.
9. Генерация (inference).
10. Метрики IS и FID.
11. Сравнение реальных и сгенерированных.
12. Траектории на фоне корта (визуализация для презентации).
13. Сборка результатов в zip.

## 7. Как запускать на Kaggle

1. Создать Kaggle Dataset из zip с картинками траекторий (внутри папки `train`
   и `test`). Подключить датасет к ноутбуку через Add Data.
2. Settings: Accelerator GPU (T4), Internet On (нужен для весов InceptionV3).
3. Сначала `OVERFIT = True` для проверки. Затем `OVERFIT = False` для полного
   прогона. Размер подвыборки задается `N_SUBSET`, число шагов - `max_steps`.
4. Run All. Результаты (чекпоинт, метрики, картинки) сохраняются в
   `/kaggle/working` и пакуются в `results_bundle.zip`.

Найти папку с данными ноутбук умеет сам: ищет директорию `train` в
`/kaggle/input`.

## 8. Ключевая идея и сложности

Главный урок проекта: качество упиралось не в размер модели, а в представление
данных и настройки обучения. Первая версия (тонкая линия поверх схемы корта) была
нерешаема для пиксельного GAN; переход к толстой сглаженной линии на белом фоне
все изменил. Среди пройденных сложностей: mode collapse (лечили соотношением шагов
1:1), полые контуры (тоньше линия плюс достаточно показов на картинку), мусорный
маркер конца (убрали кружок), недотренированность на большом наборе (счет в шагах,
а не эпохах).

Схема корта используется только на этапе визуализации (накладывается под готовую
генерацию), в обучающих данных ее нет.
