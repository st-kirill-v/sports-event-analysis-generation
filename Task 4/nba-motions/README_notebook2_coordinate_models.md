# Второй ноутбук: Coordinate-VAE

Описание ноутбука `GAN_VAE_final_notebook.ipynb` в части, которая входит в наше
сравнение - вариационный автоэнкодер на координатах (Coordinate-VAE).

Примечание: в ноутбуке есть и экспериментальные baseline (Mask-GAN, Coordinate-GAN),
включаемые флагами `RUN_MASK_GAN_BASELINE`, `RUN_COORD_GAN`. В итоговое сравнение
они не входят, мы берем из ноутбука только Coordinate-VAE.

## 1. Назначение

Координатный подход к генерации траекторий. В отличие от пиксельного DCGAN, где
траектория это картинка, здесь траектория моделируется как последовательность
координат:

```
[B, 50, 2]   (50 точек, каждая (x, y), x, y в [0, 1])
```

Модель генерирует координаты, а линия строится алгоритмически. За счет этого
связность траектории получается по построению, без разрывов.

## 2. Данные и переход к координатам

Исходные данные - те же PNG траекторий, что и у DCGAN. Из картинки извлекается
маска (фон, линия), затем маска переводится в упорядоченную последовательность
точек:

1. Берется наибольшая связная компонента.
2. Скелетизация (skimage), если доступна.
3. Поиск концевых точек и проход по скелету.
4. Если проход не удался, запасной вариант: сортировка пикселей по x.
5. Ресемплинг по длине дуги в ровно 50 точек.
6. Нормализация координат в [0, 1].

Координатный датасет кэшируется (`coord_cache_train.npy`, `coord_cache_test.npy`).
Сплит делается по `game_id`, чтобы эпизоды одного матча не утекали между train и
test.

## 3. Архитектура Coordinate-VAE

VAE работает на плоском векторе из 100 чисел (50 точек по 2 координаты).

Энкодер:

```
Linear(100, 256) -> ReLU
Linear(256, 128) -> ReLU
mu:     Linear(128, latent_dim)
logvar: Linear(128, latent_dim)
```

Репараметризация: `z = mu + sigma * eps`, latent_dim = 32.

Декодер:

```
Linear(latent_dim, 128) -> ReLU
Linear(128, 256) -> ReLU
Linear(256, 100) -> Sigmoid
reshape -> [B, 50, 2]
```

Схема - в `vae_architecture.png`.

## 4. Функция потерь

beta-VAE:

```
loss = MSE(реконструкция) + beta * KL
```

По умолчанию: `beta = 0.001`, `latent_dim = 32`, Adam `lr = 1e-3`. Лучший
чекпоинт сохраняется по validation MSE в `best_coord_vae.pt`, лог в
`coord_vae_log.csv`.

## 5. Генерация

Генерация координат: сэмплируем латентный вектор `z` из N(0, 1) и прогоняем через
декодер, получаем 50 точек. Точки рендерятся в линию и накладываются на схему
корта для визуальной оценки.

## 6. Метрики

Для координатных моделей FID менее показателен (он считается для масок), поэтому
основной упор на координатные метрики:

```
mean_start_x, mean_start_y, mean_end_x, mean_end_y
path_length_mean, path_length_std
displacement_mean, displacement_std
bbox_width_mean, bbox_height_mean
smoothness_mean, percent_out_of_bounds
```

Также для rendered масок считаются `fraction_of_broken_trajectories` и
`connected_components_count`. Для координатного подхода ожидается доля рваных
траекторий близкая к 0 и число связных компонент близкое к 1 (линия связна по
построению).

## 7. Как запускать на Kaggle

Главный конфиг в начале ноутбука. Для нашего сравнения нужен только Coordinate-VAE:

```python
SMOKE_TEST = False
FAST_DEBUG = True            # 50k train, 1k test; FULL mode при False

RUN_MASK_GAN_BASELINE = False
RUN_COORD_GAN = False
RUN_COORD_VAE = True
```

Датасет подключается через Add Data, путь ноутбук находит сам (рекурсивный поиск
PNG в `/kaggle/input`). Результаты сохраняются в `/kaggle/working`.

## 9. Ключевая идея

Координатный VAE дополняет пиксельный DCGAN: он генерирует путь как набор точек,
поэтому линия всегда связна, а обучение стабильнее (нет состязательной игры).
Сравнение двух подходов показывает сильные стороны каждого: DCGAN дает реалистичную
форму и лучший FID, Coordinate-VAE гарантирует связность и проще в обучении.
