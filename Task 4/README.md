# Task 4 - Генерация карт траекторий игроков NBA

[← К главному README](../README.md)

Task 4 исследует генерацию траекторий движения игроков NBA. Рассматриваются два подхода: пиксельная генерация карт траекторий через DCGAN и координатная генерация через VAE.

## Цель

Построить генеративную модель, которая создает реалистичные карты траекторий игроков NBA, и сравнить качество пиксельного и координатного представления данных.

## Данные

- **Источник:** NBA SportVU 2015/16 movement data.
- **Содержание:** координаты мяча и 10 игроков, 25 кадров в секунду.
- **Площадка:** 94 на 50 футов.
- **Представления:** изображение траектории `64x128 RGB` и координатная последовательность из 50 точек.

## Методы

### DCGAN

Траектория рендерится в изображение `64x128 RGB`, после чего сверточный GAN генерирует такие изображения напрямую.

Компоненты:

- generator;
- discriminator;
- adversarial training;
- визуальное сравнение real/fake;
- IS и FID через InceptionV3.

### Coordinate-VAE

Траектория моделируется как последовательность координат. Вариационный автоэнкодер учит латентное пространство и генерирует точки, которые затем рендерятся в линию.

## Запуск

### DCGAN на Kaggle

1. Собрать набор картинок:

```bash
python data/build_demo.py ... --zip
```

2. Загрузить zip как Kaggle Dataset.
3. Открыть `final-gan.ipynb` на Kaggle.
4. Подключить датасет, выбрать GPU и включить Internet для весов InceptionV3.
5. Сначала запустить режим `OVERFIT = True` для проверки на 200 картинках.
6. Затем запустить `OVERFIT = False` для полного прогона.

### Локальный smoke-запуск

```bash
python train.py
python generate.py
python evaluate.py
```

Локальный запуск предназначен для быстрой проверки pipeline; основной training выполняется на GPU.

## Сравнение подходов

| Критерий | DCGAN | Coordinate-VAE |
|---|---|---|
| Представление данных | Изображение | Координаты |
| Что генерирует | Карту траектории `64x128` | Последовательность точек |
| Тип модели | GAN | VAE |
| Дискриминатор | Да | Нет |
| Связность траектории | Средняя | Высокая |
| Сложность обучения | Высокая | Низкая |
| Метрики | IS, FID | reconstruction MSE |

## Результаты и метрики

| Подход | Метрики |
|---|---|
| DCGAN | IS `2.24`, FID `37.9` |
| Coordinate-VAE | reconstruction MSE около `5e-4` |

IS и FID используются как приближение, потому что InceptionV3 обучена на фотографиях, а траектории представлены как штриховые изображения. Поэтому важным критерием остается визуальное сходство с реальными траекториями.

## Вывод

Качество генерации сильно зависит от представления данных. Толстая сглаженная линия на белом фоне делает задачу более устойчивой для DCGAN, а координатный VAE обеспечивает связность траектории за счет самой структуры представления.
